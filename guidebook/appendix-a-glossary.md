# Appendix A: Glossary

This glossary uses practical, workflow-oriented definitions.

## A

- ADMET: Absorption, distribution, metabolism, excretion, and toxicity profile categories used for risk balancing.
- Artifact: Any file or structured output used as evidence in a workflow.
- Async mode: Execution mode where a job runs in background and status is polled later.

## B

- Baseline: Reference benchmark run used for regression comparison.
- Benchmark gate: Pass/fail decision based on defined performance and uncertainty criteria.
- Binder: Designed peptide/protein-like entity specification used in structure/design workflows.
- Bundle: Packaged regulatory evidence set with manifests and integrity checks.

## C

- Campaign: A bounded discovery objective with executable plan, outputs, and governance context.
- Checklist gate: Structured compliance or readiness review that may include automated and manual items.
- ClawCures: Campaign orchestration package that plans via OpenClaw and executes via typed tool interfaces.
- CMC: Chemistry, manufacturing, and controls domain for formulation/process/stability/release readiness.

## D

- Decision packet: Organized evidence package used in a stage-gate review.
- DMTA: Design, make, test, analyze loop used heavily in medicinal chemistry.
- Dry run: Non-destructive or low-cost validation mode for workflows.

## E

- Evidence: Reproducible outputs and metadata supporting a decision.
- Estimand: Formal definition of treatment effect and intercurrent-event handling in clinical analysis.

## F

- Fold: Structure prediction workflow for protein-ligand or binder complexes.

## G

- Gate: Formal quality, policy, or readiness checkpoint before progression.
- Governance: Practices that make decisions reproducible, reviewable, and auditable.

## H

- Hypothesis: Testable statement linking mechanism and expected outcome.

## I

- Integrity verification: Process that confirms artifact contents have not changed unexpectedly.
- Intercurrent event: Event after treatment start that affects interpretation of endpoint data.

## L

- Lead optimization: Stage where hit molecules are improved across potency, safety, exposure, and developability.
- Lineage: Graph or trace that links decisions to upstream data, tools, models, and artifacts.

## M

- Manual-review item: Checklist requirement requiring human judgment rather than full automation.
- MCP: Model Context Protocol; in this ecosystem, typed tool interfaces exposed by `refua-mcp`.
- Mission objective: High-level campaign goal that drives planning behavior.

## O

- OpenClaw: Planner endpoint used by ClawCures for structured plan generation.

## P

- Policy trace: Recorded checkpoints showing why autonomous plan decisions were accepted or rejected.
- Portfolio ranking: Prioritization process across disease programs based on multi-factor criteria.
- Preclinical workup: Planning and analysis output package for preclinical and CMC decision support.
- Provenance: Metadata describing data/model/tool/runtime origins.

## R

- Refua: Core scientific toolkit for structure, affinity, design, and properties.
- Refua Studio: Web command center coordinating campaign and lifecycle operations.
- Regression gate: Benchmark step that blocks releases when unacceptable performance drift is detected.

## S

- SAR: Structure-activity relationship; mapping structural changes to observed activity and liabilities.
- Stage-gate: Milestone review mechanism deciding proceed, hold, or rework.
- Structured tool call: Typed JSON request executed by `refua-mcp` tools.

## T

- Traceability matrix: Table linking decisions, artifacts, provenance, and gate outcomes.
- Transportability: Assessment of whether findings in one population are likely to hold in another.

## V

- VOI: Value of information; expected decision improvement from collecting additional data.
